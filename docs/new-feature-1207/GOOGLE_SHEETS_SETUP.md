# Google Sheets 資料表設定指南

**版本**: 1.0
**建立日期**: 2026-02-04

---

## 需要新增的資料表

請在您的 Google Sheets 中新增以下 3 個工作表。

---

## 1. 收貨人資訊表

**工作表名稱**: `收貨人資訊`

### 欄位結構

| 欄位 | 欄位名稱 (第一列) | 資料類型 | 必填 | 說明 |
|------|------------------|---------|------|------|
| A | 收貨人統一編號 | 文字 | 是 | 主鍵，用於匹配（如：A123456789） |
| B | 收貨人英文名稱 | 文字 | 否 | 收貨人英文名稱（如：WANG XIAO MING） |
| C | 收貨人電話 | 文字 | 否 | 收貨人聯絡電話（如：0912345678） |
| D | 海關註記 | 文字 | 否 | 有值表示有註記，匯出時標紅（如：注意、黑名單） |

### 範例資料

```
| 收貨人統一編號 | 收貨人英文名稱    | 收貨人電話   | 海關註記 |
|--------------|-----------------|------------|---------|
| A123456789   | WANG XIAO MING  | 0912345678 | 注意     |
| B987654321   | LIN MEI LI      | 0923456789 |         |
| C555666777   | CHEN DA WEI     | 0933445566 | 黑名單   |
```

### 設定步驟

1. 新增工作表，命名為「收貨人資訊」
2. 在第一列 (A1:D1) 輸入欄位名稱
3. 從第二列開始輸入資料

---

## 2. 問題件表

**工作表名稱**: `問題件`

### 欄位結構

| 欄位 | 欄位名稱 (第一列) | 資料類型 | 必填 | 說明 |
|------|------------------|---------|------|------|
| A | 貨物名稱 | 文字 | 是 | 需完全匹配的貨物名稱 |

### 範例資料

```
| 貨物名稱 |
|---------|
| 電子煙   |
| 仿冒品   |
| 毒品     |
| 槍械零件 |
```

### 設定步驟

1. 新增工作表，命名為「問題件」
2. 在第一列 (A1) 輸入「貨物名稱」
3. 從第二列開始輸入需要標記的貨物名稱

### 使用說明

- 系統會將 Excel 中的「貨物名稱」與此表進行**完全匹配**
- 匹配成功的項目會在匯出時標記為紅色背景
- 請確保名稱完全一致（包含空格、標點符號）

---

## 3. 艙單編號設定表

**工作表名稱**: `艙單編號設定`

### 欄位結構

| 欄位 | 欄位名稱 (第一列) | 資料類型 | 必填 | 說明 |
|------|------------------|---------|------|------|
| A | 設定名稱 | 文字 | 是 | 設定的唯一識別名稱（如：台北港編號） |
| B | 格式定義 | 文字(JSON) | 是 | 編號格式的 JSON 定義 |
| C | 黑名單規則 | 文字(JSON) | 否 | 排除編號的 JSON 規則 |
| D | 當前編號 | 文字 | 否 | 上次使用的最後編號 |
| E | 建立時間 | 日期時間 | 否 | 建立時間戳記 |
| F | 更新時間 | 日期時間 | 否 | 最後更新時間戳記 |

### 格式定義 JSON 結構

```json
{
  "segments": [
    { "type": "alpha", "length": 3 },
    { "type": "numeric", "length": 2 }
  ]
}
```

**說明**:
- `type`: 區段類型
  - `alpha`: 英文字母 A-Z
  - `numeric`: 數字 0-9
- `length`: 位數 (1-5)

**範例**:
- `英文3位 + 數字2位` = AAA00 → AAA01 → ... → ZZZ99
- `數字2位 + 英文2位` = 00AA → 00AB → ... → 99ZZ

### 黑名單規則 JSON 結構

```json
{
  "ranges": [
    { "start": "AAA05", "end": "AAB06" }
  ],
  "singles": ["AAA23", "BBB56"]
}
```

**說明**:
- `ranges`: 區間排除（包含起始和結束）
- `singles`: 單個排除

### 範例資料

```
| 設定名稱     | 格式定義                                              | 黑名單規則                                           | 當前編號 | 建立時間            | 更新時間            |
|-------------|------------------------------------------------------|-----------------------------------------------------|---------|--------------------|--------------------|
| 台北港編號   | {"segments":[{"type":"alpha","length":3},{"type":"numeric","length":2}]} | {"ranges":[],"singles":[]} | AAA00   | 2026-02-04 10:00:00 | 2026-02-04 10:00:00 |
| 高雄港編號   | {"segments":[{"type":"alpha","length":2},{"type":"numeric","length":3}]} | {"ranges":[{"start":"AA100","end":"AA199"}],"singles":["AB000"]} | AA200   | 2026-02-04 10:00:00 | 2026-02-04 10:00:00 |
```

### 設定步驟

1. 新增工作表，命名為「艙單編號設定」
2. 在第一列 (A1:F1) 輸入欄位名稱
3. 可以先建立一筆預設設定作為測試

---

## 驗證清單

設定完成後，請確認以下項目：

- [ ] 「收貨人資訊」工作表已建立
- [ ] 「收貨人資訊」欄位名稱正確（第一列）
- [ ] 「問題件」工作表已建立
- [ ] 「問題件」欄位名稱正確（第一列）
- [ ] 「艙單編號設定」工作表已建立
- [ ] 「艙單編號設定」欄位名稱正確（第一列）

---

## 注意事項

1. **工作表名稱必須完全一致**
   - 請確保名稱沒有多餘的空格
   - 大小寫需一致

2. **欄位名稱必須在第一列**
   - 系統會以第一列作為欄位名稱

3. **JSON 格式必須正確**
   - 艙單編號設定的「格式定義」和「黑名單規則」需要是有效的 JSON
   - 可以使用 [JSON 驗證工具](https://jsonlint.com/) 檢查格式

4. **海關註記欄位**
   - 只要有任何值就會觸發紅色標記
   - 留空表示沒有註記
